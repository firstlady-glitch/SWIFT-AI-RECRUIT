# SwiftAI Recruit - Internal Code of Conduct

As members of the SwiftAI Recruit engineering team, we are committed to maintaining a professional, respectful, and secure work environment. This Code of Conduct applies to all internal communication channels, repositories, and development activities.

## 1. Professionalism & Respect
*   **Communication:** Treat all team members with respect. constructive criticism is encouraged; personal attacks are strictly prohibited.
*   **Collaboration:** Foster a culture of mentorship and knowledge sharing. Help your colleagues succeed.
*   **Inclusivity:** We value diverse perspectives and backgrounds. Discrimination or harassment of any kind will not be tolerated.

## 2. Security & Confidentiality
*   **Data Privacy:** Never commit production credentials, API keys, or sensitive user data to the repository. Use environment variables.
*   **Trade Secrets:** Do not share internal code, architectural diagrams, or roadmap details with anyone outside the authorized team.
*   **Access Control:** Do not share your access credentials. Report any suspicious activity or security vulnerabilities immediately to the Security Team.

## 3. Engineering Excellence
*   **Code Quality:** specific adherence to the project's style guides and linting rules is required. Write clean, maintainable, and self-documenting code.
*   **Testing:** New features must include appropriate tests. Do not break the build.
*   **Review:** All code changes must undergo peer review via Pull Request before merging to the main branch.

## 4. Enforcement
Violations of this Code of Conduct may result in disciplinary action, up to and including termination of employment and legal action for breaches of confidentiality.
